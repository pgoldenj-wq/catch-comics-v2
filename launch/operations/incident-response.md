# Incident Response — the 12 launch incidents

**Trust-first containment rule: when accuracy is uncertain, hide or label the affected data. Never keep presenting it as verified while you investigate.** Wrong data is worse than missing data — that includes during incidents.

**Universal first move:** `npm run launch:smoke` (15s) tells you *which* surfaces are broken. Universal rollback: [rollback-guide.md](rollback-guide.md) (~2 min).

| # | Incident | Immediate action | Do NOT | Verify impact | Contain / roll back | Escalate when | Recovered when |
|---|---|---|---|---|---|---|---|
| 1 | **Production down** | Vercel dashboard → latest deployment status; `launch:smoke` | Don't redeploy blindly; don't touch DNS | Is it one route or all? 5xx vs timeout? | Bad deploy → instant rollback (guide). Vercel platform outage → vercel-status.com, wait | Down >15 min with announcement live → post "we're on it" on socials | smoke PASS ×2, 10 min apart |
| 2 | **Search broken** | Try `/api/search?q=saga` directly — API or page? | Don't touch `lib/search/` scoring under pressure | API 500 = server/DB; page stuck = client | Recent deploy → rollback. DB (Neon) hiccup → usually self-heals in minutes; check Neon status | Broken >30 min | Flagship queries return labelled results |
| 3 | **Product pages failing** | Open 3 different slugs — one product or all? | Don't run any DB "fix" script under pressure | All = deploy/DB. One = data row | All → rollback. One → find what's null/odd in that row, fix forward calmly | All pages >15 min | smoke product check passes |
| 4 | **Covers gone / WRONG cover** | Wrong cover = trust emergency: get the slug + screenshot | Don't run a cover backfill mid-incident | R2 outage (all covers) vs single wrong match | R2 down → letter fallbacks already honest, ride it out. Wrong cover → `UPDATE ... SET cover_image_url = NULL` for THAT row is the approved containment (single-row, reversible — record the old URL first) | Wrong covers on >1 product = stop, audit before nulling more | Correct cover or honest placeholder |
| 5 | **Wrong prices** | Identify retailer + listing; check `last_seen_at` | Don't hand-edit prices to "look right" | One retailer's feed? All? How stale? | Stale/garbage feed → deactivate that retailer (`retailers.is_active = false` — single flag, reversible, hides all its rows honestly) | Users reporting wrong prices on socials | Feed re-synced + spot-checked vs retailer site |
| 6 | **Affiliate redirects fail** | Click one `/go/` link; note status + destination | Don't edit `lib/affiliate.ts` or `/go` live | 404s (bad ids) vs 500s (code/DB) vs wrong destination (AWIN wrap) | Code → rollback. AWIN-side breakage → links still redirect unwrapped? Revenue loss ≠ user harm; fix calmly | All redirects 500 >15 min | smoke `go` check passes + manual click lands right |
| 7 | **External API spike** (eBay/CV quota burning) | Vercel logs: which route, which IPs/UA | **Don't set `RATE_LIMIT_DISABLED=1`** (that's the off switch, not the fix) | KV rate-limit 429s appearing? Legit traffic or bot? | Limits already live (price-hint 120/min, comic 200/min, ebay 40/min, search). Persistent abuser → tighten that route's limit constant, deploy | Quota exhausted (search hints/covers degrade honestly) | Route traffic normal 1h |
| 8 | **Vercel cost/traffic spike** | Vercel Usage tab: which function, what shape | Don't panic-pause the project | Real launch traffic (good!) vs bot flood (one route, weird UAs) | Bots → Vercel Firewall/challenge on that path; rate limits absorb API abuse | Spend Management alert fires (set it up NOW — founder-account-actions.md) | Usage curve matches plausible traffic |
| 9 | **Secret exposure suspected** | Identify which secret, where | Don't commit "removals" that leave it in git history | Actually a secret, or a public id? | Rotate at the SOURCE immediately (Neon / eBay / AWIN / CV / Vercel env), redeploy. GitHub secret-scanning alert = treat as confirmed | Any confirmed live secret → rotate within the hour | Old credential revoked + site works on new one |
| 10 | **Bad deploy reached prod** | Confirm which commit broke it (`git log`, Vercel deploy list) | Don't stack fix-forward commits while users see breakage | smoke + one manual page | **Instant rollback** (guide) — then fix on a branch at leisure | Always roll back first if trust-visible | smoke PASS on rolled-back prod |
| 11 | **Enrichment corrupts data** | STOP the enrichment job (Ctrl-C the terminal running it) — checkpoint makes stopping safe | **Never delete/edit checkpoint files**; don't run competing writers | Sample recently-updated rows (`updated_at DESC`) for wrong creators/covers/links | Job stays stopped; wrong CV matches → set `cv_match_suspect` on affected rows (display auto-degrades honestly) | >100 rows corrupted → full audit before restart | Sampled rows clean; job restarted from checkpoint |
| 12 | **Retailer feed stale/misleading** | `npm run launch:health` — per-retailer lastSeen + stale counts | Don't extend the 30-day threshold to hide the problem | Which retailer, how old, is stock wrong too | ≥30d rows already grey (non-Amazon) or hidden (Amazon). Whole-feed distrust → `is_active = false` for that retailer | A retailer's data wrong (not just old) | Fresh sync verified against retailer site |

**Numbers that matter:** WoB syncs daily (AWIN feed) · Amazon has NO refresh source at all — affiliate-only, stored offers age out honestly (Rainforest retired 2026-07-13, account closed) · ComicVine 200/hr shared with the enrichment job · rollback ≈ 2 minutes.
