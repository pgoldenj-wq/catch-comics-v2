# launch/operations — the launch operating system

Everything Joe needs to run Catch Comics at launch, in one place. Built Wave 3 (2026-07-13).

**One-click version:** the desktop shortcut **"Catch Comics Command Centre"** runs both commands below, starts the Mission Control server and opens it — see [COMMAND-CENTRE-LAUNCHER.md](COMMAND-CENTRE-LAUNCHER.md) (install: `create-desktop-shortcut.ps1`).

## The two commands

```bash
npm run launch:smoke    # 20 public checks against production (~15s, safe, read-only)
npm run launch:health   # read-only DB health snapshot + deltas + Amazon deadline
```

Both write `*-latest.json` here; **Mission Control** (`launch/mission-control.html`) reads them and shows verdicts with timestamps. Run both, then reload Mission Control.

## Automation

`.github/workflows/launch-smoke.yml` runs the production smoke daily at 08:00 UTC (and on demand from the Actions tab). **A failure fails the workflow → GitHub emails you.** It needs no secrets and makes ~15 polite public requests. The health check stays manual — it needs the local `.env.local` database access.

Smoke runs make one `/go` click per run with User-Agent `cc-launch-smoke/1` — filter that UA out of click analytics.
