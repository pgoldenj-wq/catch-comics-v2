# War Room — Full Product Audit · 03 Jul 2026

Branch: `polish/war-room-2026-07-02` (5 commits, not pushed). Production untouched.
Trust/UX findings from the 02 Jul production triage are incorporated by reference
(see `smoke-verdict.json` + Smoke Test V3) — this audit added the layers nothing
had covered: accessibility, performance, mobile, and code-level integrity.

## Executive summary

The biggest wins of this session were not cosmetic. The audit found and fixed:
(1) a **permanent 60fps re-render loop** on the homepage (every visitor's CPU,
battery, and jank budget, forever), (2) a **product-page crash class** — any DB
cover URL on a non-allowlisted host kills the page in dev and breaks covers in
production, with live AWIN proxy URLs already in the data — and (3) the real
root causes of three long-standing smoke-test issues that had resisted fixes
because the visible symptom pointed at the wrong layer (CC-002 was a flexbox
`min-width:auto` inflation, not a missing overflow clip; CC-024 was an inline
style defeating a responsive class; CC-008 was native `<button>` vertical
centering interacting with a data gap).

The launch gate remains **⛔ NO-GO with 9 open P0s** — but 3 of those 9 now have
fixes sitting on this branch awaiting deploy + production verification
(CC-002, CC-008, CC-024), and the remaining 6 all have implementation-ready
plans below.

## Accessibility report

**Healthy:** all images carry `alt` (9 decorative empty-alt correct); zero unnamed
buttons; carousel arrows have `aria-labels`; marquee is `aria-hidden`; related
cards have visible focus rings; `main/nav/footer` landmarks present; no horizontal
scroll at 375px; cover sways + (now) marquee + carousel drift honor
`prefers-reduced-motion`; hero chips documented at ~6.9:1 contrast.

**Gaps (backlog):** the Top Deals carousel auto-scrolls with no visible pause
control (WCAG 2.2.2 — reduced-motion helps but a pause affordance is the strict
fix, B8); hover-only interactions (hover zones, cover zoom) have no keyboard
equivalent beyond the arrows; duplicated mobile/desktop DOM doubles heading
structure (display:none keeps it out of the a11y tree — acceptable, monitor).

## Performance report

- **Fixed:** permanent 60fps whole-page re-render (worst finding; also the reason
  preview screenshots timed out — continuous animation keeps the renderer busy).
- `/api/series-preview` takes **2.1s** server-side on the homepage (B9).
- `/api/homepage-deals` 762ms.
- Cover images for CV/OL/GB hosts intentionally bypass next/image optimization
  (documented tradeoff in CVCoverImage); hero covers hotlink comicvine.gamespot.com.
- Product pages ISR 1h — note: ISR caching can mask data-driven crashes until
  revalidation (that's why prod "looked fine" while dev crashed).
- DOM ~960 nodes at mobile — healthy. Homepage renders both mobile and desktop
  DOM trees (hidden via CSS) — acceptable at current size.

## Trust report (delta over 02 Jul triage)

1. **Retailer image proxies are contaminating cover data.** AWIN productserve
   URLs (200×200, white-letterboxed) reached `cover_image_url`. Rendering them
   as covers violates the Cover Zero policy; feeding them to next/image breaks
   pages. Frontend now defends (this branch); the data must be purged and the
   writer stopped (B1 — highest-priority backlog item).
2. `/comic/[id]` legacy page still carries its own "Prices for" toggle and a
   separate layout — if reachable, it's another CC-023-class inconsistency (B7).
3. Publisher missing on some Top Deals cards (Wings of Fire, Kondo) — data gap
   now visually masked, worth a backfill query (B6).

## Backlog — implementation-ready plans

**B1 · Purge productserve cover URLs + stop the writer (P0, data-trust).**
Files: retailer sync that maps AWIN datafeed image → cover (likely
`lib/adapters/awin*` / sync scripts); one-off script mirroring
`cleanup-non-comics.ts` patterns. Steps: SQL count `cover_image_url LIKE
'%productserve%'` → null them → find the assignment site and exclude proxy hosts
→ re-run count. Risk: low (nulling falls back to designed placeholder). Verify:
count=0, affected product pages render fallback or CV fetch.

**B2 · Autocomplete dedupe + volume discovery (CC-013+CC-026, P0).**
File: `app/api/autocomplete/route.ts`. Dedupe series suggestions by
(normalised name + CV volume id), prefer primary/English publisher entry; add
local-catalogue volume matches for `<series> vol` patterns above CV series rows.
Test corpus: "Saga Vol" (expect Saga Vol 1-5, one Saga series row), "Invincible
Vol", "One Piece Vol". Risk: medium (suggestion relevance) — gate behind
side-by-side manual check. Verify in production with the CC-013 queries.

**B3 · Issue-syntax normalisation (CC-025, P0).** Files: search query
pre-normaliser in `lib/search/` before ranking; rules + full test corpus already
written in the CC-025 card ("issue 2"/"number 2"/"no. 2" → "#2"; vol/volume
untouched; don't weaken fuzzy honesty). Verify: corpus queries return the issue
as top result with weakMatch=false.

**B4 · Flagship Vol 1 ingestion sweep (CC-019, P0).** Use
`ingest:cv-series --volume-id` for JJK; audit top-50 series for missing Vol 1s
(SQL: series with volumes but no vol-1 row). Risk: low, additive.

**B5 · Creators data coverage (CC-016, P0).** Volume-level `cv_metadata.creators`
is empty — enrich from CV volume's first-issue credits as fallback during
`enrich-catalogue-cv.ts`; backfill flagship series first. UI already shipped.

**B6 · Search-card cover fallback (CC-027, P0).** Search cards render letter
placeholder when `cover_image_url` NULL while product hero falls back to CV.
Either reuse the CV-fallback path in search cards (client fetch — perf cost) or
priority re-backfill R2 covers for high-traffic titles (preferred; aligns with
Cover Zero). Pair with B1.

**B7 · Legacy `/comic/[id]` audit (P1).** Determine reachability (links from
static Top Deals fallback + old bookmarks). Either redirect to `/product/`
equivalents or align layout; it duplicates the region toggle and predates the
universal product-page system (CC-023).

**B8 · Carousel pause control (P2, WCAG 2.2.2).** Visible pause/play button on
Top Deals; state in ref, no re-render.

**B9 · series-preview endpoint 2.1s (P2).** Profile query; likely needs the same
two-stage CTE treatment as homepage-deals or a 5-min cache.

**B10 · Component test seam (P2, engineering).** Zero tests exist for the
homepage/search/product surfaces — each war-room fix was verified live instead.
Minimum: url-filters unit tests (isBadCoverUrl/canUseNextImage) + a playwright
smoke for the four fixed behaviours.

Would a collector trust it? **Closer.** The homepage no longer visually leaks or
janks; product pages no longer crash on proxy covers; the duplicated selector is
gone. But trust hinges on the remaining P0s — a collector who searches "Saga Vol"
and gets six identical Saga rows, or opens JJK and can't find Vol 1, still walks.
B1-B6 are the launch line. Fix those, re-run Smoke Test V3 sections 01-05 against
production, and the NO-GO flips honestly.
