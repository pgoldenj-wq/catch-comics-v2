How Smoke Test V3 works

The loop

Pick a section in the left rail → follow the guided steps → take a screenshot → paste it → annotate → issues are created automatically → drive them to Verified → mark the section Passed.

Screenshots — paste only

Capture with Win+Shift+S (or ⌘+Shift+4 on Mac), then press Ctrl+V anywhere on this page. The screenshot drops into the section you have open. No uploads, no file pickers. You can also drag an image straight onto the paste zone.

Annotating

Tools: V select · C circle · R rectangle · A arrow · P pencil · H highlighter · T text. Colours map to severity — Red = Critical (P0), Orange = High (P1), Yellow = Medium (P2), Blue = Question (P3), Green = Positive. Every shape gets a number and a matching issue card.

Issue workflow

Open → Diagnosed → Fix In Progress → Fixed → Ready For Verification → Verified → Closed. Set status on each card; the dashboard and launch centre update live.

Launch decision

The Launch Centre computes GO / GO WITH CONDITIONS / NO-GO from open P0/P1 issues and section completion. You can override the verdict manually before launch.

Everything is saved

All progress, screenshots, annotations and issues persist in your browser (localStorage). Use Export to pull Markdown reports for Claude.

New in V3 — Self-contained Master Report

Export → Self-Contained Master Report (HTML) generates one standalone file with everything together per section: checklist results, notes, issue cards, and the annotated screenshots embedded inline as evidence — plus statuses and verification. No separate PNGs to hunt for. Open it in any browser or print to PDF with Ctrl+P. V3 keeps its own saved data, separate from V2 — use Import from V2 on the Export screen to pull an existing V2 session in.
